C CAP Theorem / Teorema CAP Definición: Conjetura formalizada en sistemas distribuidos que demuestra la imposibilidad física y lógica de garantizar simultáneamente más de dos de las siguientes tres propiedades en un almacén de datos particionado: Consistencia Estricta (Consistency), Disponibilidad (Availability), y Tolerancia a Partición de Red (Partition Tolerance). Definition: A fundamental theorem in distributed systems stating that a distributed data store can simultaneously provide at most two out of three guarantees: Consistency (all nodes see the same data), Availability (every request receives a response), and Partition Tolerance (the system survives network drops).
C CQRS (Command Query Responsibility Segregation) / CQRS Definición: Patrón arquitectónico avanzado que divide drásticamente las operaciones de manipulación de datos (Commands: Escrituras, Actualizaciones) de las operaciones de lectura de datos (Queries). Permite optimizar, escalar y modelar de forma totalmente independiente los canales de lectura y escritura. Definition: An architectural pattern that separates the data models and execution paths for write operations (Commands) from read operations (Queries). This division allows independent optimization, scaling, and scaling architectures for both pathways.
C Concurrency Control / Control de Concurrencia Definición: Mecanismo algorítmico integrado en el motor de una base de datos para coordinar e interlinear la ejecución simultánea de transacciones que compiten por los mismos registros físicos de información, evitando fenómenos de corrupción de datos como lecturas sucias o actualizaciones perdidas. Definition: A core database management engine mechanism coordinating simultaneous transaction executions that compete for identical physical records. It prevents anomalies like dirty reads, non-repeatable reads, or lost updates using locking or multi-version concurrency control (MVCC).
