A ACID (Atomicity, Consistency, Isolation, Durability) / ACID Definición: Conjunto de cuatro propiedades métricas fundamentales que garantizan que las transacciones en un sistema de base de datos se ejecuten de forma fiable. Si alguna de estas propiedades no se cumple, la transacción completa se aborta, dejando el sistema en su estado consistente anterior. Definition: A set of four fundamental properties governing database transactions to guarantee data validity and structural integrity despite unexpected system crashes, errors, or power failures.
A API Gateway / Puerta de Enlace de API Definición: Patrón de diseño arquitectónico que actúa como un proxy inverso y único punto de entrada para los clientes en un ecosistema de microservicios. Centraliza tareas transversales como el enrutamiento de peticiones, la autenticación, la terminación SSL y el control de tasa de tráfico (rate limiting). Definition: An architectural design pattern that serves as a reverse proxy and single entry point for clients interacting with a microservices ecosystem. It centralizes cross-cutting concerns like request routing, authentication, SSL termination, and rate limiting.
A Availability / Disponibilidad (Teorema CAP) Definición: Atributo de calidad de un sistema distribuido que garantiza que cada nodo operativo que no haya fallado devuelva una respuesta legítima (no un error o un timeout) para cualquier petición que reciba, sin importar si contiene el dato de escritura más reciente de la red. Definition: A key system quality attribute within the CAP theorem ensuring that every non-failing node in a distributed system returns a non-error response for any received request, though without guaranteeing it contains the most recent write.
