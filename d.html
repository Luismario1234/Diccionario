D DBMS (Database Management System) / SGBD Definición: Sistema de software complejo que actúa como interfaz entre los usuarios/aplicaciones y las bases de datos físicas almacenadas en hardware. Gestiona de manera estructurada el almacenamiento, la recuperación de datos, la seguridad de accesos y la integridad transaccional. Definition: A comprehensive software suite that serves as the interface between software applications and physically stored data. It governs structural definitions, data retrieval optimizations, access control security, and transactional scaling.
D DDD (Domain-Driven Design) / Diseño Guiado por el Dominio Definición: Enfoque metodológico y arquitectónico de ingeniería de software que propone centrar el diseño de sistemas complejos en el modelo del negocio real (Dominio). Define fronteras lógicas y conceptuales rigurosas mediante patrones estratégicos como Bounded Contexts (Contextos Acotados) y Lenguaje Ubicuo. Definition: A software development approach focusing on centering complex system design around a deeply analyzed model of the business reality (Domain). It establishes clear conceptual boundaries via Bounded Contexts and a shared Ubiquitous Language.
D Deadlock / Bloqueo Mutuo Definición: Condición anómala y crítica de fallo en sistemas concurrentes donde dos o más transacciones se bloquean de forma indefinida entre sí. Ocurre cuando la Transacción A retiene un recurso que la Transacción B necesita, mientras que la Transacción B retiene el recurso que la Transacción A requiere para finalizar.
